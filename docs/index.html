XML + XSD validation for JavaScript, backed by Apache Xerces-C.

xerces-wasm gives Node.js and browser projects a small TypeScript API for validating XML against XSD schemas, including imported and included schema files.

Quick Start

Validate XML from strings or files.

Import validate for in-memory content or validateFiles when a Node.js workflow already has paths on disk.

From strings

import { validate } from "xerces-wasm";

const result = await validate(xmlText, xsdText);

if (!result.valid) {
  console.log(result.schemaErrors);
}

From files

import { validateFiles } from "xerces-wasm";

const result = await validateFiles(
  "./document.xml",
  "./schema.xsd"
);

Input Types

Use the data shape your app already has.

Input Supported by Typical use
string validate Editor buffers, request bodies, generated XML
Buffer validate Node.js file reads and server pipelines
Blob / File validate Browser upload flows
File path validateFiles CLI tools and build scripts

Schema Bundles

Resolve xs:include and xs:import explicitly.

Provide a root schema as entry and map each imported file by the exact schemaLocation value used inside the schema. 

const result = await validate(xmlText, {
  entry: mainXsdText,
  imports: {
    "types.xsd": typesXsdText,
    "common.xsd": commonXsdText
  }
});

API Reference

Two functions and one result shape.

validate(xml, xsd, targetNamespace?)

Validates in-memory XML against a single schema or a SchemaBundle. targetNamespace is optional and can override the namespace detected from the entry XSD. 

function validate(
  xml: XmlInput,
  xsd: XsdInput,
  targetNamespace?: string
): Promise<ValidationResult>;

validateFiles(xmlPath, xsd)

Convenience helper for Node.js file paths. The schema argument can be a single path or a bundle of paths. 

function validateFiles(
  xmlPath: string,
  xsd: string | {
    entry: string;
    imports?: Record<string, string>;
  }
): Promise<ValidationResult>;

ValidationResult

interface ValidationResult {
  valid: boolean;
  parseErrors: Diagnostic[];
  schemaErrors: Diagnostic[];
}

interface Diagnostic {
  message: string;
  line: number;
  column: number;
  severity: "warning" | "error" | "fatal";
}

Diagnostics

Parse errors and schema errors are separated.

Scenario parseErrors schemaErrors
Valid XML and valid schema [] []
Schema violations only [] One or more diagnostics
XML syntax error Fatal diagnostics May include native validator diagnostics
Schema error before syntax error Fatal diagnostics Diagnostics collected before parsing stopped

Runtime Notes

Where it fits.

Node.js

Use validateFiles for path-based workflows, or read files as Buffers and pass them to validate.

Browsers

Use validate with strings, Blobs, or Files from upload and editor workflows.

Multi-schema projects

Keep schema bundle keys aligned with the schemaLocation values inside the entry XSD.
